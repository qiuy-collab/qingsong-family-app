# 松小暖对话助手概览

## 目标

在家属端首页提供一个可拖动的松小暖 3D 悬浮助手，让用户能够：

- 通过拖动调整悬浮位置，松手后自动吸附到左右边缘
- 刷新页面后恢复上一次停留位置
- 轻点小人打开聊天面板
- 通过对话方式了解产品功能、页面用法和服务流程
- 在聊天过程中获得由后端代理大模型生成的回复

## 用户对象

- 首次进入家属端的家属用户
- 想快速了解功能入口和服务信息的老用户
- 需要通过聊天了解机构、订单、健康、动态和服务日志的用户

## 成功标准

- 首页右下角稳定展示松小暖，拖动和点击互不干扰
- 松手吸边逻辑在手机宽度下稳定可用
- 刷新页面后悬浮位置保留
- 对话面板支持多轮聊天、历史消息展示、发送中状态和失败重试
- 前端所有核心页面数据由后端接口提供
- 后端能连通本地 PostgreSQL 和外部大模型接口
- 本地可完成数据库初始化、前后端联调、端点测试和 CI 验证

## 范围

### 本次范围

- 首页松小暖悬浮助手
- 前端聊天 UI
- FastAPI + PostgreSQL 后端
- 聊天会话和消息持久化
- 本地 demo 鉴权
- 本地 PostgreSQL binaries 启动脚本
- GitHub Actions 前后端 CI

### 不在本次范围

- 真正的 3D 骨骼动画
- 正式账号体系和短信登录
- 多租户后台管理系统
- 生产环境后端部署

## 交互流程

1. 用户进入首页，松小暖默认显示在右侧底部安全区域。
2. 用户按住拖动松小暖，松手后自动吸附到左右边缘。
3. 前端把位置写入 `localStorage`。
4. 用户轻点松小暖，打开聊天面板。
5. 前端读取最近会话，若不存在则创建会话。
6. 用户发送消息，后端写入消息并调用大模型。
7. 后端保存助手回复并返回前端。
8. 前端更新聊天列表，小人继续保持说话态伪动画。
